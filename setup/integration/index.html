    Integrating with Tetra Connect¶

    This guide explains how to integrate Tetra Connect's Search Service into your Java application to perform person record searches.

    Overview¶

    Tetra Connect provides a REST API that accepts search requests and returns scored, ranked results. The API supports two response formats:

    • Protocol Buffers (recommended) - Compact binary format for optimal performance
    • JSON - Human-readable format for debugging and testing

    For Java applications, Tetra Connect provides pre-built Protocol Buffers model JARs that enable type-safe integration without manual parsing.

    Quick Start¶

    1. Add Maven Dependency¶

    Add the Protocol Buffers model library to your project:

    Maven:

    <dependency>
    <groupId>uk.co.tetra.connect</groupId>
    <artifactId>search-api-model</artifactId>
    <version>${tetra-connect.version}</version>
</dependency>

    Gradle:

    dependencies {
    implementation 'uk.co.tetra.connect:search-api-model:${tetraConnectVersion}'
}

    Accessing the JAR

    The Protocol Buffers model JAR is published to GitHub Packages. You'll need the same credentials provided for accessing container images. See the Deployment Guide for access details.

    2. Configure Maven/Gradle Repository¶

    Add GitHub Packages as a repository:

    Maven (pom.xml):

    <repositories>
    <repository>
        <id>github</id>
        <url>https://maven.pkg.github.com/tetra-connect-releases/*</url>
    </repository>
</repositories>

    Configure credentials in ~/.m2/settings.xml:

    <servers>
    <server>
        <id>github</id>
        <username>YOUR_GITHUB_USERNAME</username>
        <password>YOUR_GITHUB_PAT</password>
    </server>
</servers>

    Gradle (build.gradle):

    repositories {
    maven {
        url = uri("https://maven.pkg.github.com/tetra-connect-releases/*")
        credentials {
            username = project.findProperty("gpr.user") ?: System.getenv("GITHUB_USERNAME")
            password = project.findProperty("gpr.key") ?: System.getenv("GITHUB_TOKEN")
        }
    }
}

    Spring Boot Integration Example¶

    Configuration¶

    Create a configuration class to set up the WebClient:

    package com.example.config;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.reactive.function.client.WebClient;

@Configuration
public class TetraConnectConfig {

    @Value("${tetra-connect.search.host}")
    private String searchHost;

    @Bean
    public WebClient tetraConnectWebClient() {
        return WebClient.builder()
                .baseUrl(searchHost)
                .build();
    }
}

    application.yml:

    tetra-connect:
  search:
    host: http://search-service:1967

    Search Client¶

    Create a client class to interact with the Search Service:

    package com.example.client;

import lombok.RequiredArgsConstructor;
import org.springframework.http.MediaType;
import org.springframework.stereotype.Component;
import org.springframework.web.reactive.function.client.WebClient;
import uk.co.tetra.connect.proto.search.ProtoTetraConnectResult;

import java.math.BigDecimal;
import java.time.LocalDate;
import java.util.Optional;

@Component
@RequiredArgsConstructor
public class TetraConnectSearchClient {

    private final WebClient tetraConnectWebClient;

    public ProtoTetraConnectResult search(
            String firstName,
            String lastName,
            LocalDate dob,
            String middleNames,
            Integer maxResults,
            BigDecimal minimumScore) {

        return tetraConnectWebClient
                .get()
                .uri(uriBuilder -> uriBuilder
                        .path("/api/search")
                        .queryParam("firstName", firstName)
                        .queryParam("lastName", lastName)
                        .queryParam("dob", dob)
                        .queryParamIfPresent("middleNames",
                            Optional.ofNullable(middleNames))
                        .queryParamIfPresent("maxNoOfResults",
                            Optional.ofNullable(maxResults))
                        .queryParamIfPresent("minimumResponseScore",
                            Optional.ofNullable(minimumScore))
                        .build())
                .accept(MediaType.APPLICATION_PROTOBUF)
                .retrieve()
                .bodyToMono(ProtoTetraConnectResult.class)
                .block();
    }
}

    Service Layer Example¶

    Use the client in your service:

    package com.example.service;

import com.example.client.TetraConnectSearchClient;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import uk.co.tetra.connect.proto.search.ProtoSearchResult;
import uk.co.tetra.connect.proto.search.ProtoTetraConnectResult;

import java.math.BigDecimal;
import java.time.LocalDate;
import java.util.List;

@Slf4j
@Service
@RequiredArgsConstructor
public class PersonMatchingService {

    private final TetraConnectSearchClient searchClient;

    public List<ProtoSearchResult> findMatches(
            String firstName,
            String lastName,
            LocalDate dob) {

        log.info("Searching for person: {} {} (DOB: {})",
            firstName, lastName, dob);

        ProtoTetraConnectResult result = searchClient.search(
                firstName,
                lastName,
                dob,
                null,  // middleNames
                10,    // maxResults
                BigDecimal.valueOf(0.7)  // minimumScore
        );

        log.info("Found {} matches in {}ms",
            result.getNoOfResults(),
            result.getTotalTimeInMillis());

        return result.getSearchResultsList();
    }
}

    Ingestion Service Integration¶

    The Ingestion Service provides endpoints for triggering data refresh and sync operations. Unlike the Search Service, the Ingestion Service uses simple REST endpoints without Protocol Buffers.

    Triggering Full Refresh¶

    Rebuild all indexes from your database:

    @Service
@RequiredArgsConstructor
public class IngestionService {

    private final WebClient ingestionWebClient;

    @Value("${tetra-connect.ingestion.host}")
    private String ingestionHost;

    public void triggerFullRefresh() {
        WebClient.create(ingestionHost)
                .post()
                .uri("/api/refresh")
                .retrieve()
                .toBodilessEntity()
                .block();

        log.info("Full refresh triggered asynchronously");
    }
}

    Asynchronous Operation

    The /api/refresh endpoint returns immediately while ingestion runs in the background. Monitor the Ingestion Service logs to track progress.

    Syncing Specific Records¶

    Update specific records by their external IDs:

    public void syncRecords(List<String> externalIds) {
    WebClient.create(ingestionHost)
            .post()
            .uri(uriBuilder -> uriBuilder
                    .path("/api/sync")
                    .queryParam("externalIds", String.join(",", externalIds))
                    .build())
            .retrieve()
            .toBodilessEntity()
            .block();

    log.info("Synced {} records", externalIds.size());
}

    Example usage:

    // Sync multiple records
ingestionService.syncRecords(List.of("12345", "67890", "11111"));

    Synchronous Operation

    The /api/sync endpoint is synchronous and waits for the sync to complete before returning. Use this for small numbers of records.

    Checking Index Size¶

    Verify the number of indexed records:

    public long getIndexSize() {
    Long size = WebClient.create(ingestionHost)
            .get()
            .uri("/api/size")
            .retrieve()
            .bodyToMono(Long.class)
            .block();

    log.info("Index contains {} records", size);
    return size;
}

  search:
    host: http://search-service:1967
  ingestion:
    host: http://ingestion-service:1970

    Complete Integration Example¶

    Combine search and ingestion in a service:

    @Service
@RequiredArgsConstructor
@Slf4j
public class TetraConnectService {

    private final TetraConnectSearchClient searchClient;

    @Value("${tetra-connect.ingestion.host}")
    private String ingestionHost;

    /**
     * Trigger full data refresh
     */
    public void refreshData() {
        WebClient.create(ingestionHost)
                .post()
                .uri("/api/refresh")
                .retrieve()
                .toBodilessEntity()
                .block();

        log.info("Full refresh initiated");
    }

    /**
     * Sync specific records after updates
     */
    public void syncUpdatedRecords(List<String> externalIds) {
        WebClient.create(ingestionHost)
                .post()
                .uri(uriBuilder -> uriBuilder
                        .path("/api/sync")
                        .queryParam("externalIds", String.join(",", externalIds))
                        .build())
                .retrieve()
                .toBodilessEntity()
                .block();

        log.info("Synced {} records: {}", externalIds.size(), externalIds);
    }

    /**
     * Search for matching persons
     */
    public List<ProtoSearchResult> searchPersons(
            String firstName, String lastName, LocalDate dob) {

        ProtoTetraConnectResult result = searchClient.search(
                firstName, lastName, dob, null, 10, null);

        return result.getSearchResultsList();
    }

    /**
     * Get total indexed records
     */
    public long getIndexedRecordCount() {
        return WebClient.create(ingestionHost)
                .get()
                .uri("/api/size")
                .retrieve()
                .bodyToMono(Long.class)
                .block();
    }
}

    Best Practices¶

    Connection Pooling¶

    Configure connection pooling for optimal performance:

    import io.netty.channel.ChannelOption;
import org.springframework.http.client.reactive.ReactorClientHttpConnector;
import reactor.netty.http.client.HttpClient;
import reactor.netty.resources.ConnectionProvider;

@Bean
public WebClient tetraConnectWebClient() {
    ConnectionProvider connectionProvider = ConnectionProvider.builder("tetra-connect")
            .maxConnections(100)
            .pendingAcquireMaxCount(500)
            .build();

    HttpClient httpClient = HttpClient.create(connectionProvider)
            .option(ChannelOption.CONNECT_TIMEOUT_MILLIS, 5000);

    return WebClient.builder()
            .baseUrl(searchHost)
            .clientConnector(new ReactorClientHttpConnector(httpClient))
            .build();
}

    Timeouts¶

    Set appropriate timeouts:

    import reactor.core.publisher.Mono;
import java.time.Duration;

public ProtoTetraConnectResult search(...) {
    return tetraConnectWebClient
            .get()
            .uri(...)
            .retrieve()
            .bodyToMono(ProtoTetraConnectResult.class)
            .timeout(Duration.ofSeconds(10))  // 10 second timeout
            .block();
}

    Retry Strategy¶

    Implement retry logic for transient failures:

    import reactor.util.retry.Retry;

public ProtoTetraConnectResult search(...) {
    return tetraConnectWebClient
            .get()
            .uri(...)
            .retrieve()
            .bodyToMono(ProtoTetraConnectResult.class)
            .retryWhen(Retry.backoff(3, Duration.ofMillis(100))
                .filter(throwable -> throwable instanceof WebClientResponseException.ServiceUnavailable))
            .block();
}

    Async Operations¶

    For non-blocking operations, work with Mono directly:

    public Mono<ProtoTetraConnectResult> searchAsync(
        String firstName, String lastName, LocalDate dob) {

    return tetraConnectWebClient
            .get()
            .uri(uriBuilder -> uriBuilder
                    .path("/api/search")
                    .queryParam("firstName", firstName)
                    .queryParam("lastName", lastName)
                    .queryParam("dob", dob)
                    .build())
            .accept(MediaType.APPLICATION_PROTOBUF)
            .retrieve()
            .bodyToMono(ProtoTetraConnectResult.class);
}

    Performance Considerations¶

    Use Protocol Buffers¶

    Protocol Buffers offer significantly better performance than JSON:

    • Faster serialization/deserialization - Binary format is more efficient
    • Smaller payload size - Reduces network bandwidth
    • Type safety - Compile-time checking of message structure

    Limit Result Size¶

    Use maxNoOfResults to limit the number of results processed:

    // Only return top 10 matches
searchClient.search(firstName, lastName, dob, null, 10, null);

    Adjust Score Threshold¶

    Use minimumResponseScore to filter low-quality matches early:

    // Only return matches with score >= 0.8
searchClient.search(firstName, lastName, dob, null, null,
    BigDecimal.valueOf(0.8));
